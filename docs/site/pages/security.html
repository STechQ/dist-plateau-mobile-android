Data Encryption

Quick SDK provides multiple layers of encryption to protect data at rest and in transit.

AES-256-GCM Encryption

The SDK uses AES-256-GCM (Galois/Counter Mode) for data encryption, providing both confidentiality and authentication. This is the same encryption standard used by government agencies and financial institutions worldwide.

Feature Specification Security Level
Algorithm AES (Advanced Encryption Standard) Military Grade
Key Length 256 bits Maximum
Mode GCM (Galois/Counter Mode) Authenticated
Key Storage Android Keystore Hardware-Backed

Network Security

All network communications are secured with industry-standard protocols and additional protection mechanisms.

🔒

SSL/TLS Encryption

All network traffic is encrypted using TLS 1.2 or higher, ensuring data privacy during transmission.

📋

Certificate Pinning

Pin specific certificates to prevent man-in-the-middle attacks even on compromised networks.

🔗

URL Whitelisting

Restrict network requests to approved endpoints only, preventing unauthorized communications.

🛠

Request Signing

Sign API requests to verify authenticity and prevent request tampering.

Certificate Pinning

Certificate pinning adds an extra layer of security by ensuring your app only communicates with servers presenting specific SSL certificates. This protects against:

  • Man-in-the-middle attacks
  • Rogue certificate authorities
  • Compromised proxy servers
  • Network-level interception attempts
⚠
Certificate Rotation
When using certificate pinning, plan for certificate rotation by including backup pins and coordinating updates with your backend team.

Security Best Practices

Follow these guidelines to maximize security in your Quick SDK application.

🔒 Always Use Encrypted Storage for Sensitive Data

Never store passwords, tokens, or personal information in plain local storage. Always use the Encrypted Storage APIs which provide AES-256-GCM encryption with Android Keystore-backed keys.

🌐 Enable Certificate Pinning

Configure certificate pinning for all API endpoints to prevent man-in-the-middle attacks. Include backup pins to handle certificate rotation smoothly.

👤 Implement Session Timeout

Configure appropriate session timeouts and automatic logout for inactive users. Clear sensitive data from memory when the app goes to background.

✅ Request Minimal Permissions

Only request permissions that your application actually needs. Request them at the point of use rather than at app launch for better user trust.

🛠 Validate All Inputs

Always validate user inputs on both client and server side. Use the SDK's built-in validation features to prevent injection attacks and data corruption.
