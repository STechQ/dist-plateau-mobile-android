Encrypted Storage

For sensitive data that requires protection, Quick SDK provides encrypted storage using industry-standard AES-256-GCM encryption.

🔒
Bank-Grade Security
Encrypted storage uses AES-256-GCM with keys stored in Android Keystore, providing the same level of security used by banking applications.

Security Features

✓ AES-256-GCM Encryption
✓ Android Keystore Keys
✓ Hardware-Backed Security

  • Biometric data references

Local Storage

Local storage provides simple key-value storage for application preferences and non-sensitive data. It's accessible through JavaScript-like APIs.

⚡

Fast Access

Optimized for quick read/write operations with minimal overhead.

📁

Simple API

Familiar localStorage-like interface for easy adoption.

📊

JSON Support

Store complex objects as JSON with automatic serialization.

Operations

  • ✓

    Set Item

    Store data with a key for later retrieval.

  • ✓

    Get Item

    Retrieve previously stored data by key.

  • ✓

    Remove Item

    Delete specific items from storage.

⚠
Security Note
Local storage is not encrypted by default. Never store sensitive data like passwords or tokens in local storage. Use Encrypted Storage instead.

Choosing the Right Storage

Selecting the appropriate storage type depends on your data requirements. Use this guide to make the right choice.

Sensitive Data (Credentials, Tokens)
Use: Encrypted Storage - Always encrypt sensitive information using AES-256-GCM with Android Keystore-backed keys.
Structured Data with Queries
Use: Room Database - When you need to store relational data and perform complex queries with filtering and sorting.
Simple Preferences
Use: Local Storage - For app settings, UI preferences, and non-sensitive configuration data.
Long-term Key-Value Data
Use: Permanent Store - When you need persistent key-value storage with database organization capabilities.
Temporary Cache
Use: In-Memory - For session-specific data that doesn't need to persist after app restart.
Web Session Management
Use: Cookies - For managing authentication sessions with backend web services.
🛈
Data Minimization
Follow the principle of data minimization - only store data that is necessary for your application's functionality, and regularly clean up data that is no longer needed.
