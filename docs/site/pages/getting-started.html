Getting Started

Learn the fundamentals of Quick SDK and start building powerful mobile applications.

Introduction

Quick SDK (Plateau Studio Mobile Android SDK) is an enterprise-grade development kit that enables rapid creation of dynamic, secure, and feature-rich Android applications. Whether you're building banking apps, e-commerce platforms, or enterprise solutions, Quick SDK provides all the tools you need.

🚀

Fast Development

Build applications significantly faster with pre-built components and templates.

🔒

Enterprise Security

Bank-grade security features protect your application and user data.

🌐

Dynamic Updates

Update your app content without requiring app store releases.

SDK Architecture

Quick SDK follows a modular architecture, making it easy to understand and extend. Here's an overview of the main modules:

Quick Mobile (Core)
The central engine that orchestrates all SDK operations, manages app lifecycle, and coordinates between modules.
Quick Bridge
Enables seamless communication between JavaScript and native Android code with 186+ API methods.
Quick Components
A library of 98+ pre-built, customizable UI components following Material Design guidelines.
Network Framework
Robust HTTP networking layer with SSL pinning, caching, and comprehensive error handling.
Rendering Framework
Transforms Quick script into native Android views using Facebook's Yoga layout engine.
Storage Module
Unified data persistence with Room database, SharedPreferences, and encrypted storage options.

Project Setup

Setting up Quick SDK in your project involves the following steps:

Step 1: Add SDK Dependencies

Add the Quick SDK dependency to your project's build configuration. Your development team will provide the necessary repository access and credentials.

Step 2: Initialize the SDK

Initialize the Quick SDK in your application's startup process. The SDK requires basic configuration including:

  • Application context
  • Environment configuration (Development, Staging, Production)
  • Security settings and certificates
  • Theme preferences

Step 3: Configure Permissions

Depending on the features you use, you may need to configure the following permissions:

✓ Internet Access
✓ Camera (QR Scanner)
✓ Location Services
✓ Storage Access
✓ Biometric Auth
✓ Push Notifications
⚠
Important
Always request only the permissions your application actually needs. Quick SDK handles runtime permission requests automatically for its components.

Building Your First Application

Once the SDK is set up, you're ready to build your first Quick SDK application. Here's the recommended workflow:

  • 1

    Design Your App Structure

    Plan your application's screens, navigation flow, and data requirements before starting development.

  • 2

    Create Your First Screen

    Use Quick SDK's component library to build your first screen. Start with basic layouts and add components as needed.

  • 3

    Add Navigation

    Implement navigation between screens using the SDK's navigation components and deep linking support.

  • 4

    Connect to Backend

    Use the Network Framework to connect your app to backend services with secure HTTP communications.

  • 5

    Test and Deploy

    Test your application using the built-in testing tools and deploy to production when ready.
