Mini-Apps (SuperApp)

Build and deploy mini-applications within a single native container using the SuperApp architecture.

What are Mini-Apps?

Mini-apps are lightweight applications that run within a host "SuperApp" container. This architecture allows organizations to build an ecosystem of services accessible from a single application, reducing the need for users to install multiple apps.

📱

Single Container

Run multiple mini-apps within one native application, sharing resources and providing a unified experience.

⚡

Instant Updates

Update mini-app content without app store releases, enabling rapid feature deployment.

🔒

Isolated Environment

Each mini-app runs in isolation with controlled access to native capabilities.

💰

Payment Support

Built-in payment processing capabilities for e-commerce mini-apps.

💡
SuperApp Model
The SuperApp model, popularized by apps like WeChat and Grab, allows a single app to serve as a platform for multiple services including payments, shopping, utilities, and more.

Architecture

Quick SDK's mini-app architecture consists of several key components working together to provide a seamless experience.

SuperApp Container
The native Android application that hosts and manages all mini-apps. It provides the runtime environment, native bridge, and shared services.
WebView Runtime
Mini-apps run within optimized WebView containers, providing web-based rendering with native performance through the Quick Bridge.
Quick Bridge
The communication layer between mini-app JavaScript and native Android capabilities, with 186+ API methods available.
App Registry
Central registry managing mini-app configurations, permissions, and routing information.

Key Benefits

  • ✓

    Shared Resources

    Mini-apps share common resources like authentication, user profiles, and native capabilities, reducing redundancy.

  • ✓

    Consistent Experience

    Users enjoy a unified look and feel across all mini-apps with shared theming and navigation patterns.

  • ✓

    Centralized Management

    Administrators can manage all mini-apps from a central portal, controlling access, updates, and analytics.

Mini-App Lifecycle

Understanding the mini-app lifecycle helps in building robust applications that handle all states correctly.

State Description Actions Available
Loading Mini-app is being loaded from server or cache Show loading indicator
Initializing Resources are being initialized, bridge connected Setup event listeners
Ready Mini-app is fully loaded and interactive Full functionality available
Background User navigated away, app is paused Save state, pause operations
Resumed User returned, app is active again Restore state, resume operations
Closed Mini-app is being terminated Cleanup, save persistent data
💡
State Persistence
Use the SDK's storage APIs to persist important state data, ensuring users don't lose their progress when switching between mini-apps.

Launching Mini-Apps

Mini-apps can be launched through various methods depending on your use case.

📄

App ID Launch

Launch mini-apps using their unique identifier for direct navigation within the SuperApp.

📷

QR Code Launch

Scan QR codes to instantly open specific mini-apps with optional parameters.

🔗

Deep Link Launch

Use URL schemes to launch mini-apps from external sources or push notifications.

📁

Published Apps

Launch production mini-apps from the app registry with version management.

Launch Parameters

When launching a mini-app, you can pass various parameters:

✓ App ID
✓ Custom Parameters
✓ User Context
✓ Theme Preferences
✓ Deep Link Data
✓ Return Callback

Communication

Mini-apps communicate with the host SuperApp and backend services through well-defined channels.

Host Communication

  • 🔗

    Bridge API

    Access native capabilities through the Quick Bridge's 186+ methods for storage, networking, device features, and more.

  • 🔗

    Host Triggers

    Send events to the host app to request actions like navigation, payment processing, or authentication.

  • 🔗

    Shared Storage

    Access shared data like user profiles and preferences that persist across mini-apps.

Backend Communication

Mini-apps can communicate with their backend services through:

✓ HTTP Requests (GET, POST, PUT, DELETE)
✓ Secure SSL/TLS Connections
✓ Request/Response Caching
✓ Error Handling & Retry

Mini-App Security

Security is paramount in a multi-app environment. Quick SDK provides multiple security layers for mini-apps.

🔒

App Isolation

Each mini-app runs in an isolated environment, preventing unauthorized access to other apps' data.

🔐

Permission Control

Granular control over which bridge methods each mini-app can access.

📋

Code Signing

Mini-app packages are signed and verified before execution.

🗃

Encrypted Storage

Sensitive mini-app data is encrypted using AES-256 encryption.

Permission Scopes

Mini-apps can be granted different permission levels:

Scope Access Level Example Capabilities
Basic Limited UI components, basic storage, navigation
Standard Normal Network requests, location, camera
Extended Elevated Payments, biometrics, encrypted storage
Full Complete All bridge methods, native navigation
