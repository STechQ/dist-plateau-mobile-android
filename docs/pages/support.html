Frequently Asked Questions

Find quick answers to common questions about Quick SDK.

What Android versions does Quick SDK support?

Quick SDK supports Android 7.0 (API level 24) and above. For the best performance and access to all features, we recommend Android 10 or later.

Does Quick SDK support Huawei devices?

Yes, Quick SDK fully supports Huawei Mobile Services (HMS) in addition to Google Mobile Services (GMS). This includes Huawei Maps, Scan Kit for QR codes, and other HMS-specific features.

How do I update my mini-apps without an app store release?

Mini-apps run within WebView containers and are loaded dynamically. You can update mini-app content on your server, and users will receive the updates the next time they open the mini-app. No app store submission is required for mini-app updates.

What encryption does Quick SDK use for sensitive data?

Quick SDK uses AES-256-GCM encryption for sensitive data storage. Encryption keys are stored in Android Keystore, which provides hardware-backed security on supported devices. This is the same level of encryption used by banking applications.

Can I use custom fonts in my application?

Yes, Quick SDK supports custom font loading. You can load custom fonts and apply them throughout your application using the SDK's typography system.

How many UI components are available?

Quick SDK includes 98+ pre-built UI components organized into categories including layouts, navigation, input controls, data display, media, and more. All components follow Material Design guidelines.

Is offline functionality supported?

Yes, Quick SDK supports offline functionality through local storage options including Room Database, Encrypted Storage, and Local Storage. You can cache data locally and sync when connectivity is restored.

Troubleshooting

Common issues and their solutions.

  • 🔧

    App crashes on startup

    Ensure you've properly initialized the SDK in your Application class. Check that all required permissions are declared in your AndroidManifest.xml and that ProGuard rules are correctly configured if using code obfuscation.

  • 🔧

    Network requests fail

    Verify that your app has the INTERNET permission. If using certificate pinning, ensure your certificates are up to date and include backup pins for rotation. Check network connectivity and SSL/TLS configuration.

  • 🔧

    Components not rendering

    Ensure the Quick Bridge is properly initialized before attempting to create components. Check your component hierarchy and ensure parent containers are properly configured.

  • 🔧

    Bridge methods not working

    Verify that the WebView context is properly established and the bridge is initialized. Check the debug logs for any JavaScript errors. Ensure required permissions for specific bridge methods are granted.

  • 🔧

    Maps not displaying

    Verify that you've properly configured your Google Maps or Huawei Maps API keys. Ensure location permissions are granted when required. Check that the correct map variant (GMS or HMS) is being used for the device.
